# 턴제 배틀 게임 - 프로젝트 지침

## 목표
누가 봐도 완벽한, 반박 불가한 최고 퀄리티의 프론트엔드 과제를 구현한다.

## 개발 워크플로우 (무한 반복)

아래 사이클을 고칠 부분이 없을 때까지 무한 반복한다:

1. **도메인별 분리 구현** — 기능 단위로 나눠서 구현
2. **리팩토링** — 가독성 개선, 코드 중복 제거, 파일당 300줄 이하 유지
3. **JS 유닛 테스트** — 비즈니스 로직 테스트
4. **E2E 테스트** — 사용자 시나리오 기반 테스트
5. **리뷰** — 전체 코드 리뷰 후 개선점 발견 시 1번으로 돌아감

## 커밋 규칙
- 커밋은 나눌 수 있는 최대한 작은 단위로 쪼갠다
- 하나의 커밋 = 하나의 의미 있는 변경
- 여러 도메인/관심사가 섞이지 않도록 분리

## dev-log (개발 일지)
- `dev-log/` 디렉토리에 모든 구현 과정을 md 파일로 기록
- 기록 대상:
  - 설계 결정 및 근거
  - 프롬프팅 내용과 결과
  - 테스트 시나리오 설정 이유
  - 리팩토링 전/후 비교
  - 기술 선택 근거
  - 트러블슈팅 과정
